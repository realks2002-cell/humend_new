# Plan: 근무배정 등록 UX 개선 (shift-plus)

## 1. 배경 및 목표

### 문제점
현재 `/admin/shifts` 등록 다이얼로그는:
- 고객사/날짜/시간을 각각 수동으로 선택해야 함
- 전체 고객사 + 전체 회원 목록이 한번에 노출됨
- FCM 메시지 내용이 하드코딩되어 관리자가 커스텀 불가
- 알림 시작 시간/재발송 주기가 DB 기본값(60분전/15분간격)으로 고정
- 엑셀 등 외부 데이터로 회원 일괄 등록 불가

### 목표
관리자가 **2단계 드롭다운**(고객사→날짜)으로 빠르게 선택하고, FCM 알림 내용/타이밍을 직접 설정하며, 승인 회원 자동 표시 + 엑셀 붙여넣기로 추가 회원을 등록할 수 있게 개선.

## 2. 기능 요구사항

### FR-01: 2단계 캐스케이드 드롭다운 (고객사 → 날짜)
- **1단계**: 고객사 선택 드롭다운 (전체 clients 목록)
- **2단계**: 선택한 고객사에 등록된 공고 날짜 목록 드롭다운
  - `job_postings` 테이블에서 해당 client_id의 공고 날짜 추출
  - daily: `work_date`, fixed_term: `start_date ~ end_date` 범위
- 날짜 선택 시 해당 공고 정보(시간, 승인회원) 자동 로드

### FR-02: 시간 하드코딩 (공고 기준)
- 고객사+날짜 선택 시, 해당 공고의 `start_time`/`end_time`을 자동 채움
- 시간 입력 필드는 **읽기 전용** (disabled) — 공고에 등록된 시간만 사용
- 해당 날짜에 같은 고객사 공고가 여러 건이면 드롭다운으로 시간대 선택

### FR-03: 첫 FCM 발송 시작 시간 + 메시지 커스텀
- **발송 시작 시간 드롭다운**: 출근 전 몇 분부터 발송할지 선택
  - 옵션: 30분 전, 60분 전, 90분 전, 120분 전
  - DB 컬럼: `alert_minutes_before` (이미 존재, 기본 60)
- **FCM 메시지 입력창**: 첫 알림 메시지 커스텀
  - 새 DB 컬럼 필요: `custom_notify_message` (TEXT, nullable)
  - null이면 기존 기본 메시지 사용

### FR-04: 리피트 알림 주기 + 메시지 커스텀
- **재발송 주기 입력**: `??분` 숫자 입력 (기본 15분)
  - DB 컬럼: `alert_interval_minutes` (이미 존재, 기본 15)
- **재발송 FCM 메시지 입력창**: 미응답 회원용 메시지 커스텀
  - 새 DB 컬럼 필요: `custom_repeat_message` (TEXT, nullable)
  - null이면 기존 기본 메시지 사용

### FR-05: 승인 회원 리스트 + 상태 표시
- 고객사+날짜 선택 시, 해당 공고의 **승인된 지원자** 자동 표시
- 전체 선택/해제 토글
- 각 회원의 상태 표시 (3단계):

| 상태 | 조건 | 표시 |
|------|------|------|
| **알림 응답** | 회원이 FCM 터치 (`confirmed_at` 존재) | 🟢 ON |
| **알림 미응답** | FCM 미터치 (`confirmed_at` null) | 🔴 OFF |
| **근접** | 근무지 2km 이내 진입 (`nearby_at` 존재) | 근접 HH:MM |
| **출근** | 근무지 30m 이내 도착 (`arrived_at` 존재) | 출근 HH:MM |

- 이미 배정된 회원 (daily_shifts에 존재) → "배정됨" 뱃지, 체크 비활성화
- 미배정 회원 → 체크 가능
- **등록 다이얼로그에서는** 아직 배정 전이므로 상태 없음
- **배정 카드(shift-table 그리드)에서** 위 3단계 상태 뱃지 표시:
  - `ON/OFF` 뱃지: confirmed 여부
  - `근접 HH:MM`: nearby_at 시각
  - `출근 HH:MM`: arrived_at 시각

### FR-06: 엑셀 붙여넣기 회원 등록
- **텍스트 영역**: 엑셀에서 복사한 데이터를 붙여넣기
- **파싱 규칙**: 탭/줄바꿈 구분, 전화번호 컬럼 인식
  - 전화번호 패턴: `010-XXXX-XXXX` 또는 `01XXXXXXXXX`
- **매칭 결과 표시**:
  - 매칭 성공: 회원 이름 + 전화번호 표시, 체크 활성화
  - 매칭 실패: "미등록 회원" 표시 (빨간색)
- 매칭된 회원은 FR-05 리스트에 자동 추가/체크

## 3. DB 변경사항

### 마이그레이션: `035_shift_custom_messages.sql`
```sql
ALTER TABLE daily_shifts
  ADD COLUMN IF NOT EXISTS custom_notify_message TEXT,
  ADD COLUMN IF NOT EXISTS custom_repeat_message TEXT;
```
- 기존 컬럼 `alert_minutes_before`, `alert_interval_minutes`, `alert_max_count`는 이미 존재
- 새 컬럼 2개만 추가 (커스텀 메시지용)

## 6. 배정 카드 상태 표시 (shift-table 그리드)

기존 5단계 상태(대기/알림발송/출근예정/출근완료/노쇼)를 다음으로 변경:

```
┌─────────────────────────────────────────────────┐
│ (주)ABC물류  📍서울 강남구...   [지도] [수정]      │
│ 🕐 08:00 ~ 17:00   총 3명                       │
├─────────────────────────────────────────────────┤
│ 김철수  010-1234-5678  🟢ON   근접 07:32  출근 07:55│
│ 이영희  010-2345-6789  🔴OFF                     │
│ 박민수  010-3456-7890  🟢ON   근접 07:45         │
└─────────────────────────────────────────────────┘
```

| 뱃지 | 조건 | 색상 |
|------|------|------|
| `ON` | confirmed_at 존재 (FCM 터치함) | green |
| `OFF` | confirmed_at null (FCM 미터치) | red |
| `근접 HH:MM` | nearby_at 존재 (2km 이내) | blue |
| `출근 HH:MM` | arrived_at 존재 (30m 이내) | green |
| `노쇼` | arrival_status = 'noshow' | red destructive |

- ON/OFF는 항상 표시
- 근접/출근은 해당 이벤트 발생 시에만 표시
- 노쇼는 기존처럼 destructive 뱃지

## 7. 구현 순서

1. **DB 마이그레이션** — 커스텀 메시지 컬럼 추가
2. **Server Actions 수정** — createShift 파라미터 확장
3. **알림 함수 수정** — 커스텀 메시지 지원
4. **크론 수정** — 커스텀 메시지 읽어서 발송
5. **page.tsx 데이터 구조 변경** — 고객사별 공고 그룹핑
6. **shift-table.tsx 등록 다이얼로그 리디자인** — 전체 UI 구현
7. **shift-table.tsx 카드 상태 뱃지 변경** — ON/OFF + 근접/출근 시각
8. **엑셀 붙여넣기 파싱 로직** — 전화번호 매칭

## 8. 검증 방법

1. 고객사 선택 → 해당 고객사 공고 날짜만 드롭다운에 표시되는지
2. 날짜 선택 → 시간이 자동 채움되고 수정 불가인지
3. 알림 설정(시작시간/주기/메시지) 입력 → DB에 정확히 저장되는지
4. 크론이 커스텀 메시지로 FCM 발송하는지
5. 승인 회원 리스트 표시 + 이미 배정된 회원 비활성화 확인
6. 엑셀 붙여넣기 → 전화번호 매칭 → 회원 추가 확인
7. 배정 카드에서 ON/OFF 뱃지 + 근접 시각 + 출근 시각 정상 표시
8. 기존 수정/삭제/지도 기능이 깨지지 않는지
